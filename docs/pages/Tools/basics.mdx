# 理解 DocsGPT 工具

DocsGPT 工具是强大的扩展功能，能显著增强您的 DocsGPT 应用能力。
它们使 DocsGPT 突破从文档检索信息的核心功能，能够执行操作、
与外部数据源交互并与其他服务集成。您可以在用户界面的 DocsGPT 应用设置中
"工具"部分查找并配置可用工具。

## 什么是工具？

-   **目的：** Tools 的主要作用是在理解用户请求（由 LLM 进行自然语言处理）与执行具体操作之间建立桥梁。这可能涉及从网络获取实时数据、发送通知、运行代码片段、查询数据库或与第三方 API 交互。

-   **LLM 作为协调器：** DocsGPT 核心的大型语言模型（LLM）被设计为智能协调器。根据您的查询和可用工具（在其元数据中定义）的声明能力，LLM 决定是否需要使用工具、选择哪个工具以及传递哪些参数。

-   **面向行动的交互：** Tools 支持更动态且面向行动的交互。例如：
    * *"可再生能源的最新消息是什么？"* - 这可能会触发网络搜索工具获取最新文章。
    * *"从我们的数据库中获取客户 ID 12345 的订单状态。"* - 这可以使用数据库工具。
    * *"总结此网页内容并将摘要发送到 Telegram 的 #general 频道。"* - 这可能涉及网页抓取工具和 Telegram 通知工具的组合使用。

## 在 DocsGPT 中使用工具（用户视角）

在 DocsGPT 中与工具交互的设计遵循直观原则：

1.  **自然语言交互：** 作为用户，您通常通过自然语言查询或命令与 DocsGPT 交互。内置的 LLM 会分析输入内容，判断是否应由某个已配置的可用工具处理特定任务。

2.  **界面配置：**
    * 工具通常在 DocsGPT 应用设置的 "Tools" 版块中进行管理和配置
    * 对于需要与外部服务交互的工具（如 Brave Search、Telegram 或通过 API 工具连接的服务），您可能需要在界面设置时提供认证凭证（如 API 密钥、令牌）或特定端点信息

3.  **工具提示工程：** 虽然 LLM 会智能调用工具，但若要实现更复杂或可靠的类智能体行为，您可能需要定制系统提示。修改提示词可指导 LLM 何时以及如何优先使用或串联工具，特别是在构建需要固定执行动作序列的智能体时。详见[自定义提示词](/Guides/Customising-prompts)

## 工具进阶

掌握 DocsGPT 工具的基础知识将开启更多可能性：

* **利用 API 工具**：如需快速集成大量外部服务，请查阅 [API 工具详细指南](/Tools/api-tool)
* **开发自定义工具**：若内置工具或通用 API 工具无法满足特定需求，您可以自行开发工具。参考我们的指南 `[开发自定义工具](/Tools/creating-a-tool)`（当前为占位链接）
* **构建 AI 智能体**：工具是 DocsGPT 中创建复杂 AI 智能体的基础构建模块。通过查看 `[智能体章节/标签概念 - 链接待补充]` 了解如何组合这些工具

通过善用工具的强大功能，您可以将 DocsGPT 转变为更灵活、更主动的助手，完美适配您独特的工作流程
