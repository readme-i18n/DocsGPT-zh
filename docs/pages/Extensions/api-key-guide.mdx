# DocsGPT API 密钥指南

DocsGPT API 密钥对于希望将 DocsGPT 模型集成到外部应用（如[我们的聊天小部件](/Extensions/chat-widget)）的开发者和用户至关重要。本指南将带您完成从上传文档到理解 API 密钥相关变量的完整流程。

## 获取 API 密钥

上传文档后，您可通过图形用户界面或 API 调用获取 API 密钥：

- **图形用户界面:** 进入 DocsGPT 网页应用的设置部分，找到 API 密钥选项，点击"创建新密钥"生成密钥。
- **API 调用:** 您也可以使用 `/api/create_api_key` 端点创建新 API 密钥。详细说明请参阅 [DocsGPT API 文档](https://gptcloud.arc53.com/)。

## 理解关键变量

创建 API 密钥时，您会遇到几个关键变量，每个变量都有特定用途：

- **名称:** 为您的 API 密钥命名以便于识别。
- **来源:** 指定与 API 密钥关联的源文档，DocsGPT 将基于这些文档生成响应。
- **ID:** API 密钥的唯一标识符，可通过调用 `/api/get_api_keys` 接口查看。
- **密钥:** API 密钥本身，将在您的应用程序中用于验证 API 请求。

准备好 API 密钥后，您现在可以通过 `/api/answer` 或 `/stream` 端点将 DocsGPT 集成到应用程序中（例如 DocsGPT 小组件或其他软件）。源文档已预设于 API 密钥中，使您能在实现过程中跳过 `selectDocs` 和 `active_docs` 等字段。

恭喜您迈出了使用 DocsGPT 增强应用程序功能的第一步！
