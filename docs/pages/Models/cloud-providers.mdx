# 将 DocsGPT 连接到云端 LLM 提供商

DocsGPT 设计用于无缝集成各类云端大语言模型(LLM)提供商，让您能够访问最先进的文档问答 AI 模型。

## 通过 `.env` 文件配置

在 DocsGPT 中配置 LLM 提供商的主要方式是通过 `.env` 文件。要全面了解所有可用设置，请参阅详细的 [DocsGPT 设置指南](/Deploying/DocsGPT-Settings)。

连接到云端 LLM 提供商时，通常需要在 `.env` 文件中配置以下基本设置：

*   **`LLM_PROVIDER`**: 此设置至关重要，用于标识您希望使用的特定云端提供商（例如 `openai`、`google`、`anthropic`）。
*   **`LLM_NAME`**: 指定您希望从所选提供商使用的具体模型（例如 `gpt-4o`、`gemini-2.0-flash`、`claude-3-5-sonnet-latest`）。请参阅提供商文档以获取可用模型列表。
*   **`API_KEY`**: 几乎所有云端 LLM 提供商都需要 API 密钥进行身份验证。从您选择的提供商平台获取 API 密钥，并安全地存储在 `.env` 文件中。

## 明确支持的云端提供商

DocsGPT 为以下云 LLM 提供商提供直接简化的支持，使配置过程更加便捷。下表列出了在 `.env` 文件中需要为每个提供商配置的 `LLM_PROVIDER` 和示例 `LLM_NAME` 值。

| 提供商                     | `LLM_PROVIDER` | 示例 `LLM_NAME`          |
| :--------------------------- | :------------- | :-------------------------- |
| DocsGPT 公共 API           | `docsgpt`      | `None`                      |
| OpenAI                       | `openai`       | `gpt-4o`                    |
| Google (Vertex AI, Gemini)   | `google`       | `gemini-2.0-flash`          |
| Anthropic (Claude)           | `anthropic`    | `claude-3-5-sonnet-latest`  |
| Groq                         | `groq`         | `llama-3.1-8b-instant`      |
| HuggingFace Inference API    | `huggingface`  | `meta-llama/Llama-3.1-8B-Instruct` |
| Azure OpenAI                 | `azure_openai` | `gpt-4o`                    |

## 连接 OpenAI 兼容的云 API

DocsGPT 的灵活架构允许您连接任何提供与 OpenAI API 标准兼容的云服务提供商，这为您打开了广阔的 LLM 服务生态系统。

要连接到兼容 OpenAI 的云服务提供商，您仍需在 `.env` 文件中设置 `LLM_PROVIDER=openai`。但还需要通过 `OPENAI_BASE_URL` 参数指定所选服务商的 API 端点。通常还需按照服务商要求提供 `API_KEY` 和 `LLM_NAME`。

**DeepSeek 配置示例(兼容 OpenAI API):**

连接提供 OpenAI 兼容 API 的 DeepSeek 时，`.env` 文件可配置如下：

```
LLM_PROVIDER=openai
API_KEY=YOUR_API_KEY # Your DeepSeek API key
LLM_NAME=deepseek-chat # Or your desired DeepSeek model name
OPENAI_BASE_URL=https://api.deepseek.com/v1 # DeepSeek's OpenAI API URL
```

请务必查阅所选兼容 OpenAI 的云服务商文档，获取其特定的 API 端点、所需模型名称及认证方式。

## 添加其他云服务商支持

如需连接上文未列出或不兼容 OpenAI API 的云服务商，可扩展 DocsGPT 实现支持。在 DocsGPT 代码库中，进入 `application/llm` 目录，该目录包含现有 LLM 集成的 Python 实现文件。可参考这些文件为您目标服务商创建新模块，完成后需在 `llm_creator.py` 中注册新模块。此过程涉及编码工作，但能实现无限扩展性，支持连接任何提供开放 API 的云端 LLM 服务。
